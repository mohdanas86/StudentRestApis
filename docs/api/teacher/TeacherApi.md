# Teacher API Documentation

## 1. Get All Teachers

### Endpoint
```
GET /teachers
```

### Description
Retrieves all active teachers with their college and course information.

### Request
- **Method:** GET
- **URL:** `/teachers`
- **Authentication:** Not required
- **Headers:** 
  - `Content-Type: application/json`

### Response

**Success Response (200 OK):**
```json
{
  "success": true,
  "data": [
    {
      "teacherId": 1,
      "employeeId": "EMP001",
      "firstName": "Anas",
      "lastName": "Alam",
      "email": "anas@example.com",
      "specialization": "Java Development",
      "collegeId": 1,
      "collegeName": "IITB",
      "courseNames": ["CS101", "CS102"],
      "isActive": true,
      "createdAt": "2026-04-07T10:30:00",
      "updatedAt": "2026-04-07T10:30:00"
    }
  ],
  "message": "Teachers fetched successfully"
}
```

**No Content Response (204 No Content):**
- Returned when no active teachers are found

**Error Response (500 Internal Server Error):**
```json
{
  "success": false,
  "message": "Failed to fetch teachers",
  "error": "Database connection error"
}
```

### Status Codes
| Code | Description                          |
| ---- | ------------------------------------ |
| 200  | Successfully retrieved all teachers  |
| 204  | No active teachers found             |
| 500  | Server error while fetching teachers |

### Example
```bash
curl -X GET http://localhost:8080/teachers \
  -H "Content-Type: application/json"
```

## 2. Create a New Teacher

### Endpoint
```
POST /teachers
```

### Description
Creates a new teacher record with the provided details. Teacher must be assigned to an existing college.

### Request
- **Method:** POST
- **URL:** `/teachers`
- **Authentication:** Not required
- **Headers:** 
  - `Content-Type: application/json`
- **Body:** JSON object with teacher details

### Request Body

```json
{
  "employeeId": "EMP006",
  "firstName": "Vikram",
  "lastName": "Patel",
  "email": "vikram@example.com",
  "specialization": "Web Development",
  "collegeId": 1
}
```

### Request Fields
| Field          | Type   | Required | Description                       |
| -------------- | ------ | -------- | --------------------------------- |
| employeeId     | String | Yes      | Unique employee identifier        |
| firstName      | String | Yes      | Teacher's first name (1-50 chars) |
| lastName       | String | Yes      | Teacher's last name (1-50 chars)  |
| email          | String | Yes      | Valid email address               |
| specialization | String | Yes      | Area of expertise                 |
| collegeId      | Long   | Yes      | ID of existing college            |

### Response

**Success Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "teacherId": 6,
    "employeeId": "EMP006",
    "firstName": "Vikram",
    "lastName": "Patel",
    "email": "vikram@example.com",
    "specialization": "Web Development",
    "collegeId": 1,
    "collegeName": "IITB",
    "courseNames": [],
    "isActive": true,
    "createdAt": "2026-04-07T14:25:30",
    "updatedAt": "2026-04-07T14:25:30"
  },
  "message": "Teacher created successfully"
}
```

**Error Response (400 Bad Request):**
```json
{
  "success": false,
  "message": "Failed to create teacher",
  "error": "College not found with ID: 999"
}
```

**Error Response (400 Validation Failed):**
```json
{
  "success": false,
  "message": "Failed to create teacher",
  "error": "Email is invalid"
}
```

**Error Response (500 Internal Server Error):**
```json
{
  "success": false,
  "message": "Failed to create teacher",
  "error": "Database error occurred"
}
```

### Status Codes
| Code | Description                           |
| ---- | ------------------------------------- |
| 201  | Teacher successfully created          |
| 400  | Validation error or college not found |
| 500  | Server error while creating teacher   |

### Validations
- `employeeId`: Required, must be unique
- `firstName`: Required, 1-50 characters
- `lastName`: Required, 1-50 characters
- `email`: Required, must be valid email format
- `specialization`: Required
- `collegeId`: Required, college must exist in database

### Example
```bash
curl -X POST http://localhost:8080/teachers \
  -H "Content-Type: application/json" \
  -d '{
    "employeeId": "EMP006",
    "firstName": "Vikram",
    "lastName": "Patel",
    "email": "vikram@example.com",
    "specialization": "Web Development",
    "collegeId": 1
  }'
```

---

## Notes
- By default, newly created teachers are set as active (`isActive = true`)
- Teachers without assignments have empty `courseNames` array
- Timestamps are automatically generated by the system
